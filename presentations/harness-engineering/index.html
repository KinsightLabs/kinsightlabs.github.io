Part 1: The Basics

What is Harness Engineering?

The Problem: Autonomous AI

  • AI agents can now write code, interact with systems, and make decisions
  • But they can also:
    • Go off track
    • Make costly mistakes
    • Enter infinite loops
    • Produce inconsistent results
  • Traditional software engineering practices aren't enough

What is Harness Engineering?

Harness Engineering is the practice of building tooling and guardrails around AI agents to optimize for reliability, performance, and safety.

  • Think of it as a safety harness for autonomous systems
  • Provides structure without removing autonomy
  • Focuses on environment design, not just model prompting

Real-World Analogy

🏗️ Construction Worker

  • Skilled and capable
  • But needs safety equipment
  • Works within guidelines
  • Has tools and boundaries

🤖 AI Agent

  • Powerful and autonomous
  • But needs guardrails
  • Operates within constraints
  • Has harness engineering

Why Now?

  • AI capabilities are exploding
    • Models can now code, plan, and execute complex tasks
  • Scale demands reliability
    • Manual oversight doesn't scale to 1000s of agent runs
  • Cost considerations
    • Token usage and API costs can spiral without control
  • Trust and safety
    • Need deterministic controls for production systems

2. Architectural Constraints

Goal: Constrain the solution space to prevent chaos

  • LLM-based constraints: Guidelines in prompts and context
  • Deterministic constraints: Custom linters, validators, tests
  • Structural patterns: Enforced architectures and conventions

Key insight: Constraining options increases reliability more than adding flexibility

Architectural Constraints: Example


class ArchitecturalValidator:
    """Enforce architectural rules"""

    def validate_new_file(self, filepath, content):
        rules = [
            self.check_naming_convention,
            self.check_imports,
            self.check_directory_structure,
            self.check_dependencies
        ]

        for rule in rules:
            if not rule(filepath, content):
                return False, rule.error_message

        return True, "Valid"

    def check_naming_convention(self, filepath, content):
        # Enforce snake_case for Python files
        return filepath.name.islower()

Part 2.5: Control Theory Connection

The Mathematical Foundation

Harness Engineering = Control Theory for AI

Control theory is the study of controlling dynamical systems using feedback

Key insight: AI agents are dynamical systems that need to be controlled to achieve desired behaviors while maintaining stability

Harness engineering applies control theory principles to autonomous AI systems

Control Theory: Core Concepts

  • System: The agent and its environment
  • State: Current condition (what the agent is doing)
  • Control input: Actions/constraints applied to the agent
  • Output: Observable behavior and results
  • Feedback: Measuring output to adjust control
  • Setpoint: Desired target state or goal
  • Error: Difference between desired and actual state

Feedback Control Loop


┌─────────────┐      Control       ┌──────────┐
│  Controller │────Input/Actions───▶│  System  │
│  (Harness)  │                     │ (Agent)  │
└─────────────┘                     └──────────┘
       ▲                                  │
       │                                  │
       │         Measurements/            │
       │         Observations             │
       │                                  ▼
       │                            ┌──────────┐
       └────────Feedback────────────│  Sensors │
                                    │(Observ.) │
                                    └──────────┘

The harness observes agent behavior and adjusts constraints/context to keep it on track

Mapping Harness ↔ Control Theory

The Three Types of Control

1️⃣ Open-Loop Control

No feedback - just execute blindly

❌ Problem: Naive prompting without verification

2️⃣ Closed-Loop Control (Feedback)

Measure output and adjust based on error

✅ Solution: Self-verification loops, observability

3️⃣ Feedforward Control

Anticipate disturbances before they happen

✅ Solution: Pre-execution constraints, context engineering

PID Control for AI Agents

The classic PID controller has three components:

P (Proportional): React to current error
Example: Immediate constraint violation → Block action
I (Integral): React to accumulated error over time
Example: Garbage collection - fix drift that accumulates
D (Derivative): React to rate of change of error
Example: Loop detection - catch spiraling behavior early

PID Harness Example


class PIDHarnessController:
    def __init__(self):
        self.error_history = []  # For integral term
        self.last_error = 0      # For derivative term

    def control(self, agent_state, setpoint):
        # Calculate error
        error = self.measure_deviation(agent_state, setpoint)

        # Proportional: Current error
        p_correction = self.Kp * error

        # Integral: Accumulated error
        self.error_history.append(error)
        i_correction = self.Ki * sum(self.error_history)

        # Derivative: Rate of change
        d_correction = self.Kd * (error - self.last_error)
        self.last_error = error

        # Total control signal
        control_signal = p_correction + i_correction + d_correction

        return self.apply_correction(control_signal)

Stability: The Critical Requirement

In control theory, stability means:

  • System doesn't diverge to infinity
  • Bounded output for bounded input
  • Returns to equilibrium after disturbances

For AI agents:

  • No infinite loops or runaway behaviors
  • Predictable responses to edge cases
  • Self-correcting when off track

Lyapunov Stability for Agents

Lyapunov function: A "energy" function that decreases over time


def lyapunov_function(agent_state, goal):
    """
    Measure 'distance' from goal.
    Should decrease with each agent action.
    """
    return distance_to_goal(agent_state, goal)

# Agent is stable if V(state) decreases over time
def verify_stability(trace):
    lyapunov_values = [
        lyapunov_function(state, goal)
        for state in trace.states
    ]

    # Check if generally decreasing
    if not is_decreasing(lyapunov_values):
        alert("Agent may be unstable!")

State Space Models

Model the agent as a state machine:


class AgentStateSpace:
    """
    State space representation:
    x(t+1) = f(x(t), u(t), w(t))
    y(t) = g(x(t), v(t))

    Where:
    x = agent state (context, history, current task)
    u = control input (constraints, tools)
    w = disturbances (unexpected inputs)
    y = observable output (actions, results)

    """

        return self.state_transition(current_state, control_input)

    def is_in_safe_region(self, state):
        return self.safety_constraints.check(state)

Adaptive Control: Learning Harnesses

Adaptive control adjusts controller parameters online


class AdaptiveHarness:
    def __init__(self):
        self.constraints = default_constraints()
        self.performance_history = []

    def adapt(self, episode_result):
        """Update harness based on performance"""
        self.performance_history.append(episode_result)

        if self.detect_pattern(self.performance_history):
            # Tighten constraints if too many failures
            if failure_rate_high():
                self.constraints = self.tighten()

            # Loosen if overly restrictive
            elif false_positive_rate_high():
                self.constraints = self.loosen()

Robust Control: Handling Uncertainty

Challenge: Agent behavior is stochastic and hard to predict

Solution: Robust control techniques

  • Worst-case guarantees: Design harness for worst possible behavior
  • Safety margins: Add buffer in constraints
  • Fail-safes: Emergency stops and rollback mechanisms
  • Graceful degradation: Reduced functionality > complete failure

Model Predictive Control (MPC)

Idea: Predict future behavior and optimize ahead


class MPCHarness:
    def control(self, agent_state):
        # Simulate multiple futures
        futures = []
        for action in possible_actions:
            predicted_trajectory = self.simulate(
                agent_state, action, horizon=5
            )
            cost = self.evaluate_trajectory(predicted_trajectory)
            futures.append((action, cost))

        # Choose action with best predicted outcome
        best_action = min(futures, key=lambda x: x[1])[0]

        return best_action

Useful for complex agents with long-running tasks

Part 3: Practical Techniques

How to Implement Harness Engineering

Technique 1: Self-Verification Loops

Problem: Agents skip validation steps

Solution: Intercept before completion and force verification


class PreCompletionChecklistMiddleware:
    """Run before agent claims completion"""

    def intercept(self, agent_response):
        if agent_response.claims_complete:
            checklist = self.generate_checklist()
            return f"""
            Before marking complete, verify:
            {checklist}

            Have you completed all items? If not, continue.
            """
        return agent_response

Technique 2: Loop Detection

Problem: Agents get stuck in repetitive patterns

Solution: Track actions and intervene when loops detected


class LoopDetectionMiddleware:
    def __init__(self):
        self.action_history = []

    def track_action(self, action):
        self.action_history.append(action)

        # Detect if same file edited 5+ times
        if self.count_repeats(action) >= 5:
            return {
                "alert": "Loop detected",
                "suggestion": "Try a different approach"
            }

Technique 3: Reasoning Budget Allocation

Goal: Optimize compute/cost vs. task complexity

"Reasoning Sandwich" Pattern

  • 🧠 High reasoning: Planning phase
  • ⚡ Low reasoning: Execution phase
  • 🧠 High reasoning: Verification phase

LangChain used "xhigh-high-xhigh" allocation to reach Top 5 on Terminal Bench

Technique 6: Tool Access Control

Goal: Limit what agents can do


class HarnessedAgent:
    def __init__(self, allowed_tools):
        self.allowed_tools = allowed_tools

    def execute_tool(self, tool_name, params):
        if tool_name not in self.allowed_tools:
            return {
                "error": f"Tool {tool_name} not allowed",
                "available": self.allowed_tools
            }

        # Additional validation
        if not self.validate_params(tool_name, params):
            return {"error": "Invalid parameters"}

        return self.tools[tool_name].execute(params)

Observability & Feedback Loops

Control theory principle: You can't control what you can't measure

Key metrics to track:

  • Success rate: % of tasks completed correctly
  • Token efficiency: Tokens per successful completion
  • Latency: Time to completion
  • Error patterns: Common failure modes
  • Loop frequency: How often agents get stuck (stability)
  • Constraint violations: Rules broken per run

These metrics form your feedback signal for control

The Feedback Loop Pattern


def harness_improvement_loop():
    while True:
        # 1. Deploy agent with current harness
        results = run_agent_on_tasks(batch_size=100)

        # 2. Analyze failures
        failures = filter(lambda r: not r.success, results)
        patterns = analyze_error_patterns(failures)

        # 3. Identify harness gaps
        gaps = identify_missing_constraints(patterns)

        # 4. Update harness
        update_context(gaps.context_needs)
        update_constraints(gaps.constraint_needs)
        update_tools(gaps.tool_needs)

        # 5. A/B test improvements
        evaluate_improvement()

A Step-by-Step Guide

Step 1: Define Your Requirements

Start by asking:

  • What is the agent's primary task?
  • What are the success criteria?
  • What are the risks or failure modes?
  • What constraints must be enforced?
  • What context does the agent need?

Step 2: Build Context Layer


def build_context_layer():
    return {
        # Static documentation
        "guidelines": load_guidelines(),
        "examples": load_examples(),

        # Dynamic environment info
        "file_structure": map_directory_tree(),
        "available_tools": list_tools(),

        # Task-specific context
        "task_requirements": get_task_spec(),
        "evaluation_criteria": get_success_criteria(),

        # Constraints
        "forbidden_patterns": load_antipatterns(),
        "required_patterns": load_best_practices()
    }

Step 3: Implement Constraints


class ConstraintLayer:
    def __init__(self):
        self.validators = [
            StyleValidator(),
            SecurityValidator(),
            PerformanceValidator(),
            ArchitectureValidator()
        ]

    def validate(self, agent_action):
        for validator in self.validators:
            result = validator.check(agent_action)
            if not result.valid:
                return {
                    "allowed": False,
                    "reason": result.reason,
                    "suggestion": result.fix_suggestion
                }
        return {"allowed": True}

Step 4: Add Safety Nets


class SafetyLayer:
    def __init__(self):
        self.loop_detector = LoopDetector()
        self.resource_monitor = ResourceMonitor()
        self.verification_gate = VerificationGate()

    def wrap_agent(self, agent):
        # Monitor during execution
        agent.on_action(self.loop_detector.track)
        agent.on_action(self.resource_monitor.check)

        # Intercept before completion
        agent.on_complete(self.verification_gate.verify)

        return agent

Step 5: Deploy & Observe


class HarnessedAgent:
    def run_with_observability(self, task):
        with trace_context() as trace:
            try:
                # Provide context
                context = self.context_layer.build()

                # Execute with constraints
                result = self.agent.execute(task, context)

                # Validate result
                validated = self.constraint_layer.validate(result)

                # Log metrics
                trace.log_metrics({
                    "success": validated.success,
                    "tokens": trace.token_count,
                    "time": trace.duration
                })

                return validated
            except Exception as e:
                trace.log_error(e)
                self.analyze_failure(trace)

Step 6: Iterate Based on Data

  1. Run agent on test tasks
  2. Collect failure traces
  3. Identify patterns in failures
  4. Add missing constraints/context
  5. Measure improvement
  6. Repeat

Key principle: Let failures guide harness evolution

Part 6: Best Practices

Do's and Don'ts

✅ Do's

  • Start simple - Add constraints incrementally
  • Make constraints explicit - Document what and why
  • Use deterministic validation - Don't rely only on LLM
  • Measure everything - Observability is critical
  • Iterate based on data - Not intuition alone
  • Balance autonomy and control - Don't over-constrain

❌ Don'ts

  • Don't over-engineer upfront - Build harness iteratively
  • Don't only prompt - Use code-based guardrails too
  • Don't ignore failures - Each failure teaches you something
  • Don't assume one harness fits all - Different tasks need different harnesses
  • Don't forget cost - Monitor token usage and latency

Common Antipatterns

❌ Vague constraints
"Write good code" → ✅ "Functions must be < 50 lines, complexity < 10"
❌ No feedback loops
Deploy and hope → ✅ Deploy, measure, analyze, improve
❌ Only LLM-based validation
Trust agent's self-assessment → ✅ Run tests, linters, validators
❌ Reactive harness building
Fix issues as they arise → ✅ Design harness based on risk analysis

Measuring Harness Effectiveness

Key questions to evaluate your harness:

  1. Are fewer runs failing over time?
  2. Is token efficiency improving?
  3. Are loops detected before they spiral?
  4. Is output quality more consistent?
  5. Are you catching errors before production?
  6. Can you explain why failures happen?
